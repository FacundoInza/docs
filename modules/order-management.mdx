## Order lifecycle

Every P2P order moves through a defined set of states. Use this reference to understand what each status means in your panel.

| Status | Description |
|---|---|
| **Pending Payment** | The order has been accepted and is waiting for the counterparty to submit payment. |
| **Paid (Awaiting Release)** | The buyer has marked the order as paid. Verification is in progress before crypto is released. |
| **Completed** | Payment has been confirmed and the order is fully settled. |
| **Cancelled** | The order was cancelled before completion, either by a party or by timeout. |
| **Under Appeal** | A dispute has been raised. The order is under review by Binance or your compliance team. |

## Multi-account view

View orders from all linked Binance accounts in a single panel. Use the built-in filters to narrow your view by account, status, date range, asset, or payment method — so your team can immediately focus on the right subset of orders without any manual sorting.
